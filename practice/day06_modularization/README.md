# Day06 模块化拆分与结构体整理

## 今日目标
将前几天的 GPIO、按键、串口、定时器功能进行初步模块化整理，并引入 `SensorData` 结构体统一管理系统数据。

## 今日内容
- 新建 LED 模块，封装 LED 开、关、翻转函数
- 新建按键模块，封装按键检测函数
- 新建数据模块，定义 `SensorData` 结构体
- 在主循环中通过结构体管理系统状态
- 让 `main.c` 只保留主流程逻辑

## 核心知识点
- 模块化拆分的目的
- `.h` 与 `.c` 的分工
- 结构体在项目中的作用
- 统一数据管理思想
- 主流程与底层驱动分离

## 实验结果
- 工程可以正常编译运行
- LED 和按键逻辑已独立封装
- 已使用 `SensorData` 结构体统一管理部分状态数据
